Goal: to develop data structure layer of VBB algorism

VBB algorism is not a specific algorism, but represents the whole running mechanism of voting based on blockchain.

Through python, we have implemented the basic data entity and encryption algorism, which mainly contains the following parts.

Part1: the object of blockchain and block

If we have votes data like this:

We will mine a blockchain as below.

Firstly, it generates a genesis block with the requirements predefined. Then it will mine other four blocks which contains the voting records and clear the voting pool, for we stipulate that the minimum length of blockchain is 3 (including the genesis block). For every two hours, if there are voting records still in the voting pool, it continue mining.

Nonce is calculated by a method that we proposed as “representative proof of work”. We hidden the algorism to calculate it, while it remains a relative low energy consumptions to make the chains more secure, which depends on the difficulty.

We also implemented the algorism to calculate the root of Merkle tree that consists of the hash value of various voting records.

Part 2: encryption

We have implemented the asymmetric encryption algorism.

  1. method that generates the public key and private key
  1. methods that encrypt the content using public key and decrypt the content using private key

​ 3.methods that generate the signature using private key and verify the signature using public key

Considering that there may be a different demand in the future for the data layer, we may adjust and update the underlying mechanism and algorism timely.
