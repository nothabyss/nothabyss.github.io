Milestone 4

Date: 2024.04.27

Primary Focus

The primary objective of Milestone 4 is to implement the voting functionality, marking a pivotal progression in our project’s development

Frontend Development

Vote Manage Page

We developed a vote management page, which allows users to start new votes and edit previous votes, and has a simple search vote function to make it easier to find a vote directly by its name, and the votes are set to different types, where the different numeric types represent the number of choices supported by the vote, (multiple or single choice).

Voting Page

Users can choose to enter a vote directly to view specific information on the voting page, you can see the information has the option to vote and the current number of votes, you can choose the object to vote, after the completion of the vote, you can see the number of votes after the update, it should be noted that because of the previous page on the type of voting settings, in the page to complete a certain number of votes, the other options of the voting button will become unavailable to complete the function of multiple choice or single choice.

Backend Development

Logic of the voting function

The development of the back-end voting functionality has been completed. The system first allows users to create voting campaigns, each of which is stored in the vote database table.
Users can choose whether the voting campaign is single- or multiple-choice, and can also provide start time, end time, and description and other related information.
In addition, each voting activity can have multiple candidates, which are stored in the voteDetail table, where vote_id is a foreign key representing which voting activity the candidate belongs to,
and the table also stores the name, description, number of votes,
start time, end time and other related information of the candidate. Users can vote for specific candidates. The specific database table structure is shown below:

Back-end API

Our voting API provides comprehensive functionality for managing polls, including the essential operations of creating, updating, deleting, and retrieving individual polls.
In addition to these core functions, our API also offers a powerful pagination feature, allowing users to efficiently navigate through large sets of poll data. With pagination,
users can specify the desired page size and retrieve subsets of poll results, ensuring optimal performance and usability even when dealing with extensive datasets.
Whether you’re a developer integrating voting capabilities into your application or a user interacting with polls, our API delivers a seamless experience with robust features to meet your needs.
